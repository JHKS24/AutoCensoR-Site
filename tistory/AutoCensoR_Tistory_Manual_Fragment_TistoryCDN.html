01

AutoCensoR가 하는 일

AutoCensoR는 이미지 한 장이 아니라 폴더 단위로 일합니다. 폴더를 열면 안의 이미지를 목록으로 불러오고, 모델을 로드한 뒤 일괄 처리를 실행하면 보호할 후보 영역을 찾아 한 번에 가립니다. 자동 결과가 완벽하지 않은 부분만 사람이 직접 손봅니다.

  • 폴더 통째로 처리한 폴더의 이미지를 한 번에 검토·처리합니다.
  • 필요한 곳만 보정자동이 놓친 부분만 브러시·영역 도구로 마무리합니다.
  • 결과 자동 저장출력 폴더에 접미사를 붙여 자동으로 저장됩니다.
솔직한 안내: 완벽한 자동 검열을 보장하지는 않습니다. 자동이 후보를 빠르게 잡아 주고, 마지막 판단과 마무리는 사람이 합니다. 원본은 함부로 덮어쓰지 않고 출력 폴더에 따로 저장합니다.
02

빠른 시작

처음 쓰는 분은 아래 다섯 단계만 따라 하면 됩니다. 각 단계는 뒤에서 더 자세히 다룹니다.

  1. 이미지 폴더 선택입력 폴더 버튼으로 이미지 폴더를 선택하면 목록이 채워지고 ‘대기 중’으로 표시됩니다.
  2. 모델 로드모델 로드 버튼으로 호환되는 .pt 모델을 불러옵니다. 로드 전에는 자동 감지가 동작하지 않습니다.
  3. 일괄 처리일괄 시작(F5) 버튼으로 폴더 전체를 처리합니다.
  4. 수동 보정브러시 검열, 복구/지우개, 영역 감지, 스탬프 도구, 퀵마스크로 필요한 곳만 다듬습니다.
  5. 저장 확인결과는 작업 중 자동으로 출력 폴더에 저장됩니다. 따로 저장 버튼을 누를 필요가 없습니다.
03

설치와 빌드: CPU Only vs Full

배포본은 두 가지입니다. 처음이라면 CPU Only를 권합니다. 가장 호환성이 좋아 어떤 PC에서도 잘 켜집니다. NVIDIA 그래픽카드가 있고 속도를 높이고 싶다면 Full을 선택하세요.

빌드대상특징
CPU Only
가장 호환성 좋음		모든 Windows PC
(그래픽카드 무관)		CUDA 드라이버 없이 실행됩니다. 속도는 느릴 수 있지만 다른 PC에서도 재현성이 가장 좋습니다. 먼저 이 빌드로 시작하길 권합니다.
Full
NVIDIA 환경		NVIDIA GPU 보유 PCCUDA 가속을 고려한 배포본입니다. 처리 속도가 빠를 수 있지만 그래픽 드라이버와 PyTorch 호환성의 영향을 받습니다.
압축을 풀고 실행 파일을 그대로 실행하는 방식입니다. 모델 가중치는 포함되어 있지 않으며, 사용자가 직접 호환 .pt 모델을 모델 로드 버튼으로 불러옵니다(5번 항목 참고).
04

권장 사양

아래는 일반적인 기준입니다. CPU Only 빌드는 그래픽카드가 없어도 동작하므로 가장 넓은 환경에서 쓸 수 있습니다.

항목CPU Only (가장 호환성 좋음)Full (NVIDIA)
운영체제Windows 10 / 11 (64-bit)Windows 10 / 11 (64-bit)
그래픽카드필요 없음NVIDIA GPU · VRAM 8GB 이상 권장
메모리(RAM)16GB 이상 권장16GB 이상 권장
CPU4코어 이상 (6~8코어 권장)4코어 이상
속도느릴 수 있음 (가장 안정적)지원 GPU에서 더 빠를 수 있음
여유 공간대량 처리는 RAM 32GB 이상과 충분한 SSD 여유 공간을 권장합니다.
먼저 CPU Only로 시작하세요. 잘 동작하는 것을 확인한 뒤, 속도가 필요하고 NVIDIA 환경이라면 Full 빌드로 옮기면 됩니다.
05

모델 로드와 .pt 모델

우측 상단의 모델 로드 버튼을 누릅니다. 로드 전에는 ‘로드 안 됨’으로 표시되고, 자동 감지가 동작하지 않습니다. 호환되는 .pt 모델을 불러오면 자동 감지가 준비되어 일괄 처리를 시작할 수 있습니다.

모델 가중치 파일은 앱에 포함되어 있지 않습니다. 사용자가 직접 호환 .pt 모델을 준비해 모델 로드 버튼으로 불러오세요. 모델 파일 위치 안내는 배포 페이지의 모델 가이드를 참고하면 됩니다.
06

폴더 작업 흐름

입력 폴더 버튼으로 폴더를 선택하면, 폴더 안 이미지가 왼쪽 목록에 채워지고 ‘대기 중(PENDING)’으로 표시됩니다. 좌측 탭(전체 · 검열됨 · 정상 · 대기 중 · 실패)으로 상태별로 추려 볼 수 있습니다.

07

일괄 처리

일괄 시작(F5) 버튼을 누르면 폴더 전체를 한 번에 처리합니다. 감지된 영역에 선택한 보호 방식(모자이크·블러·흑백·단색)이 적용되고, 하단에 처리 완료 개수와 소요 시간이 표시됩니다.

08

보호 방식

단색 · 흑백 · 블러 · 모자이크 네 가지를 같은 보호 영역에 적용합니다. 방식 선택은 “어떻게 가릴지”만 정합니다. 방식을 바꿔도 감지 범위나 영역 크기는 변하지 않습니다.

09

감지 민감도

감지 민감도는 후보를 얼마나 넓게 잡을지 정하는 임계값입니다. 보호 방식과는 다른 설정입니다.

핵심: 값이 낮을수록(예: 0.01) 더 민감하게, 더 넓게 후보를 잡습니다. 값이 높을수록 확실한 후보만 남습니다.
임계값후보 영역언제 쓰나
0.01 (낮음)가장 넓게 · 가장 민감놓치는 영역을 줄이고 싶을 때 (이후 수동 정리)
0.20 ~ 0.50권장 시작 범위대부분의 경우 여기서 시작
0.50 (높음)확실한 후보만과다 검출을 줄이고 싶을 때

가장 좋은 값은 사용하는 모델과 이미지에 따라 달라집니다. 0.20~0.50에서 시작해 결과를 보며 조금씩 맞추세요. 너무 많이 잡히면 값을 올리고, 놓치는 곳이 있으면 값을 내린 뒤 불필요한 부분을 복구/지우개로 정리하면 됩니다.

11

자동 저장과 출력

처리와 보정 결과는 작업을 진행하는 동안 자동으로 적용·저장됩니다. 출력 폴더에 접미사를 붙여 저장되며, 별도의 수동 저장 단계를 누를 필요가 없습니다. 원본은 그대로 두고 결과만 출력 폴더에 따로 저장됩니다.

참고: 출력 폴더, 파일명 접미사, 품질은 설정 → 저장에서 조정합니다.

13

문제 해결

자동 감지가 동작하지 않아요.

모델이 로드되었는지 확인하세요. 로드 전에는 ‘로드 안 됨’으로 표시되고 자동 감지가 동작하지 않습니다. 호환되는 .pt 모델을 모델 로드 버튼으로 불러온 뒤 다시 시도하세요.

Full 빌드가 잘 켜지지 않아요.

Full 빌드는 그래픽 드라이버와 PyTorch 호환성의 영향을 받습니다. 가장 호환성이 좋은 CPU Only 빌드로 먼저 동작을 확인해 보세요.

처리가 너무 느려요.

CPU Only 빌드는 속도가 느릴 수 있습니다. NVIDIA GPU(8GB 이상 권장)가 있다면 Full 빌드에서 더 빠를 수 있습니다. 한 번에 처리하는 이미지 양을 줄이는 것도 도움이 됩니다.

너무 많이 / 적게 잡혀요.

감지 민감도를 조정하세요. 값이 낮을수록(예: 0.01) 더 넓게 잡고, 높일수록 확실한 후보만 남습니다. 그 뒤 브러시 검열과 복구/지우개로 마무리하면 됩니다.

과하게 가린 부분을 되돌리고 싶어요.

복구/지우개(X)로 일부만 되돌리거나, 현재 이미지 초기화(W)로 한 장을 처음 상태로 되돌릴 수 있습니다.

결과 파일이 어디에 저장됐는지 모르겠어요.

결과는 출력 폴더에 접미사를 붙여 자동 저장됩니다. 출력 폴더와 파일명 접미사는 설정 → 저장에서 확인·변경할 수 있습니다.

01

What AutoCensoR does

AutoCensoR works on a whole folder, not one image at a time. Open a folder, load a model, run a batch, and it finds candidate regions to protect and covers them in one pass. You only step in for the parts the automatic pass didn't get right.

  • Process a folder at onceReview and process all images in a folder together.
  • Fix only what's neededTouch up what automation missed with brush and region tools.
  • Output saved automaticallyResults are written to an output folder with a suffix.
An honest note: it does not guarantee perfect automatic censoring. Automation finds candidates quickly; the final judgment and clean-up are yours. Originals are never overwritten, and results go to a separate output folder.
02

Quick start

New here? Just follow these five steps. Each one is explained in more detail below.

  1. Choose an image folderOpen an input folder with the Input Folder button; the list fills in and items show as “pending”.
  2. Load a modelLoad a compatible .pt model with Load Model. Detection does not run before a model is loaded.
  3. Run batch processingPress Start Batch (F5) to process the whole folder.
  4. Correct manuallyUse Brush censor, Restore/eraser, Region detect, Stamp tool, and Quick mask for the final touch-up.
  5. Confirm savingResults are saved to the output folder automatically while you work, with no separate save button.

Installation & builds: CPU Only vs Full

There are two builds. If you're starting out, choose CPU Only: it's the most compatible and starts reliably on any PC. Pick Full only if you have an NVIDIA graphics card and want more speed.

BuildForNotes

05

Model loading & the .pt model

Press the Load Model button at the top right. Before loading it shows “not loaded” and detection does not run. Once a compatible .pt model is loaded, detection becomes ready and you can start batch processing.

Model weight files are not bundled with the app. Prepare your own compatible .pt model and load it with Load Model. For where to place the model file, see the model guide on the downloads page.
06

Folder workflow

Open an input folder with the Input Folder button, and its images fill the list on the left, marked “PENDING”. Use the left-side tabs (All · Censored · Clean · Pending · Failed) to filter by status.

07

Batch processing

Press Start Batch (F5) to process the whole folder at once. Your chosen protection style (mosaic, blur, grayscale, or solid) is applied to detected regions, and the bottom bar shows the completed count and elapsed time.

08

Protection styles

Solid, grayscale, blur, and mosaic are all applied to the same protected region. The style only decides “how to cover it”. Changing the style never changes the detected range or region size.

09

Detection sensitivity

Detection sensitivity is a threshold that decides how widely candidates are caught. It is a separate setting from the protection style.

Key point: lower values (e.g. 0.01) are more sensitive and catch a wider set of candidates. Higher values keep only the confident ones.
ThresholdCandidate regionsWhen to use
0.01 (low)Widest · most sensitiveTo miss fewer regions (then clean up by hand)
0.20 to 0.50Recommended starting rangeA good default for most cases
0.50 (high)Confident candidates onlyTo reduce over-detection

The best value depends on your model and images. Start at 0.20 to 0.50 and adjust as you watch the results. If too much is caught, raise it; if regions are missed, lower it and tidy up extras with restore/eraser.

11

Auto-save & output

Processing and correction results are applied and saved automatically as you work. They are written to the output folder with a suffix, so there is no separate manual save step. Originals are left untouched; only the results go to the output folder.

Note: output folder, filename suffix, and quality are adjusted under Settings → Save.

Troubleshooting

Detection isn't running.

Make sure a model is loaded. Before loading it shows “not loaded” and detection does not run. Load a compatible .pt model with Load Model and try again.

The Full build won't start properly.

The Full build depends on graphics driver and PyTorch compatibility. Confirm things work with the most compatible CPU Only build first.

Processing is very slow.

The CPU Only build can be slow. With an NVIDIA GPU (8GB+ recommended), the Full build can be faster. Processing fewer images at a time also helps.

Too much / too little is being caught.

Adjust the detection sensitivity. Lower values (e.g. 0.01) catch more widely; higher values keep only confident candidates. Then finish with brush/restore.

I want to undo an over-covered area.

Use restore/eraser (X) to undo part of it, or reset current image (W) to return one image to its original state.

I can't find where the output was saved.

Results are auto-saved to the output folder with a suffix. Check or change the output folder and suffix under Settings → Save.

Vague feedback is hard to act on, while specific reproduction steps and expected behavior make the app easier to improve.
